`@hareru/mcp` is an MCP (Model Context Protocol) server that exposes Hareru UI's design tokens, schema, and component registry to AI agents via stdio transport.

## Installation

```bash
npm install @hareru/mcp
```

## Claude Desktop Configuration

Add to `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "hareru-ui": {
      "command": "npx",
      "args": ["-y", "@hareru/mcp"]
    }
  }
}
```

Or if installed locally:

```json
{
  "mcpServers": {
    "hareru-ui": {
      "command": "node",
      "args": ["./node_modules/@hareru/mcp/bin/hareru-mcp.mjs"]
    }
  }
}
```

## Resources

| URI | Description |
|-----|-------------|
| `hareru-ui://tokens` | DTCG design tokens (light/dark themes, 129 CSS custom properties) |
| `hareru-ui://tokens/schema` | JSON Schema — CSS variable enums and type constraints |
| `hareru-ui://components` | Component registry — 41 components with CVA variants and Props |

## Tools

| Tool | Input | Output |
|------|-------|--------|
| `get-component-usage` | `componentName: string` | Markdown: import, variants table, props, JSX example |
| `validate-token-value` | `tokenType: string, value: string` | Valid/Invalid with pattern description |

## Prompts

| Prompt | Parameters | Description |
|--------|-----------|-------------|
| `create-ui` | `description`, `framework?` | Generates a UI component using Hareru UI conventions |

## Programmatic Usage

```typescript
import { createServer } from '@hareru/mcp';

const server = createServer();
await server.connect(transport);
```

## CSS Exports

| Export path | Contents |
|-------------|---------|
| `@hareru/tokens/css` | CSS variables (colors, spacing, typography tokens) |
| `@hareru/ui/styles.css` | Convenience bundle (tokens + reset + components — one import) |
| `@hareru/ui/styles.layer.css` | Convenience bundle wrapped in `@layer hui` |
| `@hareru/ui/styles/components.css` | Component styles only (portable — pair with `@hareru/tokens/css`) |
| `@hareru/ui/styles/components.layer.css` | Component styles wrapped in `@layer hui` (Tailwind v4 coexistence) |
| `@hareru/ui/styles/baseline.css` | Minimal reset (box-sizing, font: inherit) — optional |
| `@hareru/ui/styles/scope.css` | `.hui-root` typography scope helper (portable opt-in) |
| `@hareru/ui/styles/animations.css` | Shared keyframes (for per-component usage) |
| `@hareru/ui/styles/components/*.css` | Per-component CSS (49 files) |

## MCP Apps Compatibility

Hareru UI works in iframe sandboxes because:

1. **CSS variables are scope-complete** — All styling uses `--hui-*` custom properties. Injecting `@hareru/tokens/css` provides all design tokens.
2. **`data-theme` works on any root** — Theme switching works on any element, not just `document.documentElement`.
3. **No CSS-in-JS runtime** — Static CSS files with BEM classes.
4. **No framework runtime conflicts** — Standard React with no required global providers.
